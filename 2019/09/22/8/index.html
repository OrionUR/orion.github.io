git笔记

2019-09-22   |   Tech   |  

本篇是关于使用git的笔记

安装Git

由于我目前只使用Linux，所以这里只记录Linux安装Git的方法
直接在terminal上输入git，如果你没安装Git，就会有提示信息，告诉该怎么安装
如：

Git命令

git init：创建仓库
git add <file>：添加file到暂存区
git commit：提交到仓库

提交到仓库，需要输入备注信息，最好如实输入，好方便以后管理

git push：提交到GitHub上（如果你已经绑定了GitHub）
git status：查看仓库当前的状态
git diff <file>：对比file文件，跟上次提交到仓库里的，修改了什么内容
git log：查看日志

如果觉得输出信息太多，可以加上--pretty=oneline参数

git reset --hard HEAD^：HEAD为当前版本，HEAD^为之前一个版本，上上一个版本就是HEAD^^

如果是往上100个版本，可以写成HEAD~100
当然也可以用commit id来代替
如果上一个版本commit id为1094adb，那么就可以用1094adb代替HEAD^

git relog：记录了每一次命令，可以用此命令查看过去的commit id
git checkout -- <file>：把file文件恢复到之前的版本

这里有两种情况
(1) xxx修改后还没被放到暂存区，那么现在就会恢复到和仓库版本一模一样的状态
(2) xxx已经添加到暂存区，但又接着修改，那么现在就会回复到和暂存区版本一模一样的状态
如果修改后又git add添加到暂存区，但还没git commit
那么可以用git reset HEAD <file>把暂存区的修改撤销掉（unstage）
再用上边的方法恢复

git rm <file>：把file文件从仓库中删除

要commit后才能确认删除，删错了可以用checkout恢复

GitHub

1. 创建SSH Key

在用户主目录里，查看是否有id_rsa和id_rsa.puh这两个文件

ls ~/.ssh/

如果没有，则创建SSH Key

ssh-keygen -t rsa -C “your email@example.com”
引号里填写的是你的Email地址

一路回车即可，如果提示你输入密码，可以不输入
若是不幸输入了，后边会教你怎么免密码登陆（我就是如此）
如果一切顺利，可以用上边查看命令看看.ssh目录里，是否有id_rsa和id_rsa.puh这两个文件
这两个就是SSH Key的密钥对，id_rsa是私钥，不能泄露出去
id_rsa.puh是公钥，可以放心地告诉任何人

2. 登陆GitHub，打开Account settings——SSH Keys页面

然后，点Add SSH Key，填上任意title，在Key文本框里粘贴id_rsa.puh文件的内容

3. 把本地仓库的内容，推送到GitHub仓库

在本地仓库目录下运行

origin后边的，是我的GitHub仓库，你要替换成自己的，否则你关联的就是我的仓库
至于右边那串东西怎么知道
从你创建的repository里，有个绿色的Clone or download选项
点开后选择右上方的Use SSH
就能看到属于你仓库的SSH Key了

由于远程仓库是空的，所以第一次推送master分支时，需要加上-u参数

此后，每次本地提交，就可以使用git push origin master推送最新修改

4. SSH警告

第一次使用push命令链接GitHub时，会得到一个警告
输入yes回车即可
如果之前有设置密码，那么会提示输入密码，直接输入即可

可以使用ssh-add ~/.ssh/id_rsa命令配置免密登陆
这样以后push就不会再要求输入密码了
