Mac下安装以及管理软件

2016-05-17   |   Tech   |  

OS X的系统跟Windows完全不一样，首先体现在软件的安装以及管理上边

在Windows上安装软件首先需要去找那个软件的安装包
然后再下载下来选择安装的地方
而OS则并不需要那样
一般来说安装软件人们可能会选择去AppStore去下载安装
但实质上，由于AppStore上的审核时间过长，生态圈不完整，限制太多以及维护成本过高
让很多应用开发者被迫离开
很多软件已经出到很后的版本，但在App Store上，还是老版本

于是，brew和brew-cask应运而生
诚然，在命令终端上安装以及管理软件对很多人（尤其是妹纸）来说并不是那么的友好
不过，这也因此提供了些机会给学计算机的理工男（承诺只修电脑，绝不过夜，欢迎来电咨询）

在安装brew以及brew-cask之前，必须先安装Xcode
现在Xcode的安装直接去App Store搜索安装即可，很方便
完成后打开terminal
输入ruby -e "$(curl -fsSL https://raw.github.com/mxcl/homebrew/go)"
就会开始自动安装brew
结束后再输入brew tap phinze/homebrew-cask && brew install brew-cask
也是开始自动安装brew-cask
这样一来brew和brew-cask也就安装完毕。

brew和brew-cask的区别，其实就是面向对象的不同
由于brew是下载源码解压然后配置以及安装，同时回包含相关依存库
并且自动配置好各种环境变量，而且易于卸载
因此brew对于程序员来说简直就是福音
简单的指令，就能快速安装和升级本地的各种开发环境
而brew-cask是下载已经编译好了的应用宝（.dmg/.pkg）
然后解压并且放在统一的目录中（/opt/homebrew-cask/Caskroom）
省去自己去寻找，下载，解压，拖拽等一系列步骤，并且卸载同样容易且干净
同时还能找到很多AppStore上没有或者最新的版本的软件
因此一般用户使用brew-cask安装软件即可
如果brew-cask上找不到想要的程序再去brew上找。

安装命令什么的十分简单
输入brew cask install xxx就可以安装xxx软件
至于要查询这个软件是否在brew-cask上，可以输入brew cask search xxx
这样brew-cask会自动搜寻库中与xxx相匹配的软件
之后输入brew cask info xxx可以查看该软件的详细信息，包括版本，是否已安装
卸载软件也十分简单，输入brew cask uninstall xxx就可以很好地卸载
想查看已经装了什么软件，可以输入brew cask list来查看
常用的命令就以上几个，还有一些其他的命令，可以直接输入brew cask来查看
至于brew的命令，其实跟brew-cask是一样的，把brew替换掉brew cask即可

至于软件的更新，目前只有两种方法

  1. 软件自己的更新选项
  2. 使用brew cask uninstall xxx && brew cask install xxx先卸载再重新安装来实现更新软件

    第二种方法去更新例如QQ这样的软件是否会导致聊天记录之类的历史纪录丢失
    我没去测试过（懒得测试）
    不过现在大部分的软件都有搜索更新的功能，可以实现自动更新，因此也不必太纠结
