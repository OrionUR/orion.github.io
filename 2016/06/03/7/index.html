捣鼓Sublime Text 3

2016-06-03   |   Tech   |  

之前一直用的是Sublime Text 2，由于搜索的时候看到很多人说3好
于是打算捣鼓一下

本体

首先遇到的问题是，brew-cask里没有Sublime Text 3
上网搜了下，找到了解决方法

然后问题又出现了
不知道为什么一直显示
Warning: Unexpected method 'appdir' called on Cask sublime-text3.
和
Error: Download failed on Cask 'sublime-text3' with message: Download failed: https://download.sublimetext.com/Sublime%20Text%20Build%203114.dmg
根据提示重新安装brew-cask并且更新brew

再然后重新安装brew cask install sublime-text3成功

插件

用Sublime Text 3最主要的也是因为它有非常丰富的插件
插件的安装一般都依赖于它本身的Package Control
关于如何安装具体可以参考官方文档
这里我简单说一下
菜单栏view -> Show Console，底部会有个命令输入框，将下边的代码复制粘贴到输入框内回车即可

安装好Package Control后，就可以使用快捷键command + Shift + p来安装插件
在命令框内输入pci能快速进入插件浏览界面

由于我现在用的也不是很久，也就引用一些大家都觉得比较好的插件
ConvertToUTF8

支持GBK, BIG5, EUC-KR, EUC-JP, Shift_JIS 等编码的插件
由于ST默认是使用UTF-8编码，但不支持其他类似GBK，CJK编码
有了这个插件，在打开对应ST不支持的编码文件的时候，会自动选择合适的编码转换

DocBlockr

DocBlockr可以自动生成PHPDoc风格的注释
它支持的语言有Javascript, PHP, ActionScript, CoffeeScript, Java, Objective C, C, C++

Emmet

快速生成html代码段的插件，强大到无与伦比，可以超快速编写HTML/CSS/JS
web开发必备插件
这里推荐两个链接：使用Emmet加速Web前端开发，Emmet 插件使用教程（转载）

SideBarEnhancements

侧边栏增强插件
这么说吧，原先ST的侧边栏就是A罩杯，有了这个插件就变成了H罩杯
默认ST不打开侧边栏，可以到View -> Side Bar -> Show Open File
之后可以通过快捷键command + k command + b来显示或者隐藏侧边栏

过多的插件会导致ST变慢变卡，请谨慎选择插件
至于删除插件，在Package Control中输入remove package
然后选择要卸载的插件即可
